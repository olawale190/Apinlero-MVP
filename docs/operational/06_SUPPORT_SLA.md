## 5. Service Level Agreement (SLA)

### 5.1 Platform Uptime

| Metric | Commitment |
|--------|------------|
| **Monthly Uptime** | 99.5% |
| **Scheduled Maintenance** | Max 4 hours/month |
| **Maintenance Window** | Sundays 02:00-06:00 GMT |
| **Maintenance Notice** | 48 hours minimum |

### 5.2 Uptime Calculation

```
Uptime % = ((Total Minutes - Downtime Minutes) / Total Minutes) × 100
```

**Exclusions from Downtime:**
- Scheduled maintenance (with proper notice)
- Issues caused by customer actions
- Third-party service outages (Stripe, WhatsApp, etc.)
- Force majeure events
- Beta/preview features

### 5.3 Service Credits

| Monthly Uptime | Credit (% of monthly fee) |
|----------------|---------------------------|
| 99.0% - 99.5% | 10% |
| 95.0% - 99.0% | 25% |
| 90.0% - 95.0% | 50% |
| Below 90.0% | 100% |

**Credit Claim Process:**
1. Submit claim within 30 days of incident
2. Email: billing@apinlero.com
3. Include: dates, times, impact description
4. Credits applied to next billing cycle
5. Maximum credit: 100% of monthly fee

### 5.4 Status Page

**URL:** status.apinlero.com

**Information Provided:**
- Current system status
- Scheduled maintenance
- Incident history
- Performance metrics

**Subscribe to Updates:**

## 6. Support Process

### 6.1 Submitting a Support Request

**Required Information:**
| Field | Description |
|-------|-------------|
| **Subject** | Brief description of issue |
| **Priority** | Your assessment (we may adjust) |
| **Description** | Detailed explanation |
| **Steps to Reproduce** | How to recreate the issue |
| **Expected Behaviour** | What should happen |
| **Actual Behaviour** | What actually happens |
| **Screenshots/Videos** | Visual evidence (if applicable) |
| **Browser/Device** | Your technical environment |

**Example Good Request:**
```
Subject: WhatsApp orders not appearing in dashboard
Priority: High

Description:
Since yesterday (15 Dec), WhatsApp orders from customers
are not appearing in my order dashboard. Customers confirm
messages are sent, but I see nothing.

Steps to Reproduce:
1. Customer sends order via WhatsApp
2. Customer receives confirmation
3. Order does not appear in Àpínlẹ̀rọ dashboard

Expected: Order appears within 1 minute
Actual: Order never appears

Browser: Chrome 120
Last working: 14 Dec around 6pm
```

### 6.2 Support Ticket Lifecycle

```
SUBMITTED ──► ACKNOWLEDGED ──► IN PROGRESS ──► RESOLVED ──► CLOSED
    │              │               │              │            │
    ▼              ▼               ▼              ▼            ▼
  Auto         Response        Updates        Solution      Feedback
  ticket #      email          provided       confirmed      survey
```

### 6.3 Escalation Process

| Level | Timeframe | Triggered By |
|-------|-----------|--------------|
| **Level 1** | Initial | New ticket |
| **Level 2** | After SLA time | Auto-escalation or customer request |
| **Level 3** | After 2× SLA time | CS Manager review |
| **Level 4** | Critical issues | Founder involvement |

**How to Escalate:**
- Reply to ticket with "Please escalate"
- Email escalation@apinlero.com
- Call (Growth plan only)

## 7. Customer Responsibilities

### 7.1 To Receive Support

Customers agree to:
- Provide accurate, complete information about issues
- Respond to requests for additional information promptly
- Attempt basic troubleshooting (clear cache, try different browser)
- Keep account contact information current
- Report issues in a timely manner

### 7.2 Acceptable Use

Support is for:
- Platform functionality issues
- How-to questions
- Bug reports
- Feature requests
- Account management

Support is NOT for:
- General business consulting
- Third-party software help
- Custom development work
- Training (beyond included onboarding)
